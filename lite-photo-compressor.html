Privacy Policy for Lite Photo Compressor

Application Identifier: com.neurontslabs.litecompressor • Effective Date: July 12, 2026

1. Media Isolation & Processing Boundaries

Lite Photo Compressor is engineered from the ground up to operate as a completely localized, high-performance image quantization pipeline. Your images, photographs, and documents represent deeply sensitive personal data; therefore, our architecture enforces strict offline containment. Pixel parsing, dimension shifting, and file size optimizations occur entirely within your device's isolated application memory sandbox.

Absolute Isolation Policy: Lite Photo Compressor features zero remote network hooks for handling media. No images selected by you or processed by our engine are ever transferred to, cached by, or visible to Neuron TS Labs or any third-party server infrastructure.

2. Storage Infrastructure & Media Access Disclosures

To safely scan, read, and write compressed media bundles across various Android API levels, the application interfaces with specific platform storage frameworks. Here is how your data behaves:

Android Framework / Permission Operational Scope Data Security Boundary
MediaStore API / Scoped Storage Enables the single compression utility and the Pro Whole Gallery Deep Scanner to map spatial media files (JPG, PNG, WEBP) directly inside device boundaries. Read access is limited strictly to files you select or explicitly target for scanning. The application never modifies or destroys your original master image files.
System Share Sheet Integration Allows users to instantly forward images from native Android galleries straight into the compressor pipeline without initializing the core app window first. The inbound URI stream is consumed ephemerally in RAM, processed directly to your exact target file size, and saved locally to the system downloads index.

3. Foreground Processing & Ambient Hardware Subsystems

During heavy batch executions or full device gallery passes, Lite Photo Compressor utilizes a high-priority Android Foreground Service to ensure execution stability. In compliance with Google Play Developer Policies, the service operates under these strict architectural rules:

  • Neon Hardware Brightness Controller: During lengthy background tasks, the app can drop system window brightness parameters down to an absolute minimum of 0.01f to reduce thermal output and minimize battery drain. This is managed locally in memory and restores instantly upon completion.
  • Thermal Guard Protection: Monitors device thermal registers in real-time. If the device temperature crosses high safe boundaries, processing algorithms automatically throttle execution speed to ensure hardware safety.
  • Screen Awake Control: Temporarily bypasses standard display timeout flags during active compression runs to block Android's battery-saving system from interrupting deep file calculations.

4. Monetization Frameworks, Purchases, and Diagnostics

To support ongoing independent software updates, Lite Photo Compressor integrates standard secure billing components and advertising SDKs. These systems are sandboxed from your actual media operations:

A. Google Play Billing System

When upgrading to the Pro tier (activating the Whole Gallery Compressor), transaction cycles are processed entirely by Google Play's native secure checkout engine. Neuron TS Labs retains anonymous purchase tokens and validation statuses locally to toggle application states, but we never capture or store credit cards, bank parameters, or identity details.

B. Diagnostic & Ad Identifiers

Standard advertising services (like Google AdMob) may collect device analytics, advertising IDs, and coarse region coordinates to manage ad frequency capping and combat fraudulent interactions. These networks are restricted by native container policies and cannot inspect your internal file libraries or look at processed document outputs.

5. User Data Rights & Subsystem Wipes

Because your image data is never uploaded to an external server, there is no remote cloud asset ledger for you to request deletion from. All persistent user configurations, premium status validation tokens, and localized session parameters reside exclusively within your physical storage container.

You have full data control. A total clean-slate wipe can be executed instantly at any time by wiping the app's cache and local files via your device settings or by completing a standard app uninstallation.
